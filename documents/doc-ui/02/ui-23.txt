6. 	Добавим подсветку корзины зеленым цветом в момент перетаскивания шлема:
	- Options - activeClass - добавим класс highlight (рамка: 2px сплошная зеленая)
	
	activeClass: 'highlight'
	
Корзина-приемник:

	
7.	Рамка не добавляется. Смотрим таблицу стилей. На корзину накладываются стили дважды:
	по идентификатору  #mycart 	и по классу  .highlight
	Тогда браузер будет действовать по приоритету - а приоритет за идентификатором.
	Зададим это css-правило не с помощью идентификатора, а с помощью класа:
	Добавим в коде блока корзины к id="mycart" еще и одноименный класс.  
		<div id="mycart" class="mycart">  </div>
		
	Исправим в таблице стилей селектор по идентификатору на селектор по классу
		.mycart
	
	Теперь рамка подсвечивается. Хотя правило для рамки прописано дважды,
	браузер возьмет то, которое находится ниже.

П О Л К А - приемник:

8.	Теперь добавим возможность вернуть шлем из корзины обратно на полку.
	То есть приемником должна стать не только корзина, но и полка. 
	Применяем метод droppable для полки. И добавляем внутрь действие drop:, 
	Для этого копируем код для корзины-приемника в полку-приемник. 
	Изменяем имя переменной helmet на helmet2. Вносим изменения для appendTo:
	Перемещение в корзину заменяем перемещением на полку.

Полка-приемник:	

		
9.	ПРОБЛЕМА: Тащим шлем в корзину - ее рамка подсвечивается. Тащим шлем 
	обратно из корзины - ее рамка тоже подсвечивается, а не должна.
	Подсветка срабатывает на попытку перетаскивать любой шлем, даже из корзины.
	Исправление вносим в корзину-приемник, в опции accept:
	Эта опция выбирает, что именно можно добавлять в корзину:
	Отберем только те шлемы, которые лежат на полке.
	
	accept: '#forHelmets div[id*=helmet]',

	activeClass:	-	для подсветки рамки полки с помощью класа
